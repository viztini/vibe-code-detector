## vibe code detector

This is a lightweight Python utility designed to identify code or documentation that may have been generated by Large Language Models (LLMs). It uses a weighted scoring system based on common "AI fingerprints," such as specific conversational patterns, structural artifacts, and "too-perfect" formatting.

---

### Features

* **Heuristic-Based Scoring:** Analyzes files for em-dashes, emojis, and polite conversational phrasing often used by AI.
* **Structural Analysis:** Detects Markdown code fence leftovers (e.g., ` ```python `) and instructional boilerplate.
* **Documentation Density:** Flags files with an unusually high ratio of comments/docstrings to actual code.
* **Format Checking:** Identifies "too perfect" formatting, such as the complete absence of trailing whitespace.
* **Multi-Language Support:** Scans Python, JavaScript, C++, Java, Go, Rust, and many other extensions.
* **Recursive Scanning:** Can scan individual files or entire directories while automatically ignoring common folders like `.git` and `node_modules`.

---

### Installation

Ensure you have Python 3.7+ installed. No external dependencies are required, the script uses the Python Standard Library.

1.  Save the script as `detector.py`.
2.  (Optional) Make it executable (Linux) :

    ```bash
    chmod +x ai_detector.py
    ```

---

### ## Usage

Run the script from your terminal by providing a path to a file or directory.

#### Basic Scan
```bash
python ai_detector.py /my_project
```

#### Detailed Scan (Verbose)
Use the `-v` or `--verbose` flag to see exactly which markers triggered the score for each file.
```bash
python ai_detector.py path/to/file.py -v
```

---

### Scoring Logic

The detector assigns a score from **0 to 100**. The higher the score, the more likely the code was AI-generated.

| Score Range | Confidence |
| :--- | :--- |
| **70 - 100** | **HIGH**: Likely AI-generated. |
| **50 - 69** | **MEDIUM**: Suspected AI involvement. |
| **30 - 49** | **LOW**: Likely human with some AI-like traits. |
| **0 - 29** | **MINIMAL**: Typical human-written code. |

### Limitations

* **False Positives:** Highly disciplined developers who use strict documentation standards (Google/Sphinx) and clean formatting may receive higher scores.
* **False Negatives:** Highly customized or "lazy" AI prompts that omit docstrings and conversational filler may pass undetected.
* **Text-Based only:** This tool analyzes static patterns and does not use machine learning or external APIs.
